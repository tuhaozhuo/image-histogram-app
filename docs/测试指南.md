# 测试指南（面向测试人员）

本指南给做验收/测试的同学。分**黑盒测试**（直接用 app 操作观察，不用写代码）和**白盒验证**（用工具精确对拍，可选）。

---

## 0. 前提：核心统计用哪个实现

- app 界面显示的「**核心耗时**」= **多线程标量 double**（精确实现），**不是 NEON**。
- **NEON** 只在「三实现性能对比」按钮里出现，是对比展示，不参与主流程。
- 实时相机用灰度平面版（相机 YUV 的 Y 通道直接统计）。

测性能/准确性时，主流程看的就是那个绿色的「核心 X ms」。

---

## 1. 功能测试（黑盒，用 app）

| # | 操作 | 预期结果 |
|---|---|---|
| F1 | 点「相册选择」选一张照片 | 显示原图预览 + 256×100 黑白直方图 + 核心耗时 + 图像尺寸 |
| F2 | 点「拍照」拍一张 | 同上（真机测；模拟器相机可能不稳） |
| F3 | 点「内置渐变图」 | 预览显示彩色渐变图，直方图为**平顶梯形**，耗时达标 |
| F4 | 点「内置真实样张」 | 预览显示荷花照片，直方图为**参差尖峰**，耗时达标 |
| F5 | 点「三实现性能对比」 | 出现对比表：单线程/多线程/NEON 三行，各有耗时、加速比、精度 |
| F6 | 点「实时相机直方图」→ 允许相机权限 | 相机预览 + 底部直方图**实时跟着画面变化**，每帧显示核心耗时 |

---

## 2. 性能测试（核心评分点：≤ 300ms）

| # | 操作 | 判定 |
|---|---|---|
| P1 | 对多张不同尺寸的图分别计算，看核心耗时 | 每次都显示绿色「**达标（≤300ms）**」 |
| P2 | 用较大的图（如手机原图 12MP） | 仍达标 |
| P3 | 三实现对比表 | 多线程比单线程快数倍；真机上 NEON 通常最快 |
| P4 | 实时相机 | 每帧核心耗时远低于 300ms，直方图流畅不卡 |

> **注意**：性能以**真机 release** 为准（模拟器/debug 有虚拟化开销，耗时偏高但功能一致）。

---

## 3. 准确性测试

### 3.1 快速形态检查（黑盒，肉眼）

直方图形态应符合图像明暗分布：

| 图像 | 直方图应呈现 |
|---|---|
| 内置渐变图（灰度均匀分布） | 平顶梯形 |
| 真实照片 | 参差尖峰（多峰多谷） |
| 偏亮的图（雪景/白墙） | 集中在**右侧**（高灰度） |
| 偏暗的图（夜景/阴影） | 集中在**左侧**（低灰度） |
| 纯色图 | 只有一根柱（该颜色对应的灰度级）为满高 |

形态明显不符 = 有问题。

### 3.2 精确对拍（白盒，用工具，需能跑命令）

**灰度化穷举验证**（最强，全覆盖 1677 万种像素）：
```bash
clang++ -O2 -std=c++17 -ffp-contract=off native/tests/verify_gray.cpp -o /tmp/vg && /tmp/vg
# 预期：标量 vs 标准公式 0 处不一致 ✓
```

**第三方金标准对拍**（对具体图片，与 numpy 独立复算逐 bin 比对）：
```bash
pip install numpy pillow
cmake -S native -B native/build -DCMAKE_BUILD_TYPE=Release && cmake --build native/build
./native/build/bench --dump 待测图.png > ours.csv
python3 native/tests/golden_check.py 待测图.png --compare ours.csv
# 预期：逐 bin 对拍：256/256 一致，0 处不同 → PASS ✓
```

（方法论详见 [开发手册.md](开发手册.md) 第 9 节。）

## 4. 边界与鲁棒性测试

| # | 操作 | 预期 |
|---|---|---|
| B1 | 纯黑图 | 不崩溃；bin0 满高、其余为 0 |
| B2 | 纯白图 | 不崩溃；bin255 满高、其余为 0 |
| B3 | 超大图（如 24MP） | 不崩溃、仍达标 |
| B4 | 选取后立刻切换/退出 | 不崩溃、无卡死 |
| B5 | 拒绝相机权限后进实时相机页 | 显示相机错误提示，不崩溃 |

bench 内置了纯黑图、空图（防除零）的自动用例：`./native/build/bench` 会打印这些用例的 PASS/FAIL。

## 5. 双端一致性测试

| # | 操作 | 预期 |
|---|---|---|
| C1 | 同一张图分别在 Android、iOS 上计算 | 直方图形态一致、耗时都达标 |

## 6. 验收标准（对齐需求文档）

- [ ] 上传、统计、归一化、绘制、耗时显示全流程可用（F1–F6）
- [ ] 典型图像核心耗时 ≤ 300ms 且正确显示（P1–P4）
- [ ] 灰度化/统计准确，与标准公式一致（3.2 层验证）
- [ ] 直方图分辨率 256×100、黑白显示（F3/F4 形态）
- [ ] 不同尺寸图稳定运行、无崩溃（B1–B5）
